# 兼容性矩阵

## 1. 目标

本文件用于说明 Mirexs 在不同操作系统、Python 版本、GPU 环境和外部依赖组合下的兼容状态。它是部署评估、采购评估和故障排查的重要依据。

## 2. 操作系统兼容性

| 能力 | Windows 10/11 | Ubuntu 20.04/22.04 | macOS 12/14 |
|---|---|---|---|
| 基础 Python 运行 | 支持 | 支持 | 支持 |
| 本地 GGUF 推理 | 支持 | 支持 | 支持 |
| CUDA GPU 加速 | 支持 | 支持 | 不支持 |
| 3D 渲染与交互 | 支持 | 支持 | 部分能力受限 |
| 语音输入输出 | 支持 | 支持 | 支持 |
| Neo4j 等外部服务接入 | 支持 | 支持 | 支持 |
| 系统自动化 | 支持 | 支持 | 部分权限和系统接口受限 |

## 3. Python 版本建议

| Python 版本 | 状态 | 说明 |
|---|---|---|
| 3.9 | 最低兼容 | 仅建议用于保守环境 |
| 3.10 | 支持 | 较稳定 |
| 3.11 | 推荐 | 当前开发优先版本 |
| 3.12 | 谨慎使用 | 某些依赖和二进制包可能存在兼容风险 |

## 4. GPU 兼容性

| GPU 环境 | 状态 | 说明 |
|---|---|---|
| 无 GPU | 可运行 | 仅适合小模型或非推理密集场景 |
| RTX 20 系列 | 可用 | 基础本地推理 |
| RTX 30 系列 | 推荐 | 兼顾性能与成本 |
| RTX 40 系列 | 强烈推荐 | 更适合多模型和高并发 |
| AMD ROCm | 部分支持 | 需要单独验证具体依赖 |
| Apple Silicon | 部分支持 | 更适合轻量和 CPU/Metal 路线 |

## 5. 外部服务兼容性

| 服务 | 角色 | 状态 |
|---|---|---|
| Neo4j | 知识图谱 | 推荐 |
| Redis | 缓存/状态辅助 | 推荐 |
| PostgreSQL | 用户与业务数据 | 可选 |
| SQLite | 本地轻量模式 | 可选 |
| InfluxDB / Prometheus | 指标与监控 | 可选但推荐 |

## 6. 使用方式说明

- `支持`：可作为正式目标环境
- `部分支持`：需补充配置和额外验证
- `可运行`：能工作，但体验和能力可能明显受限

## 7. 兼容性验证建议

当新增依赖、模型或平台适配时，应至少验证：

- 依赖能否安装
- 配置能否加载
- 核心入口能否导入
- 关键链路能否运行
- 性能是否在可接受范围内
