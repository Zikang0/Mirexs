# 安全规范

## 1. 目标

本规范定义 Mirexs 在开发和部署阶段必须满足的安全底线，包括数据保护、输入防护、访问控制、审计和事件响应。

## 2. 安全目标

- 保护用户隐私和敏感数据
- 防止恶意输入、越狱和越权访问
- 确保关键行为可审计、可追踪、可归责

## 3. 数据分级

- 公开数据：可公开展示的信息
- 受限数据：普通用户数据、配置和交互记录
- 敏感数据：身份信息、生物特征、密钥、令牌、支付或健康相关数据

## 4. 核心安全域

### 4.1 输入安全

- 参数校验
- 敏感词和高风险模式识别
- 工具调用白名单或权限校验

### 4.2 访问控制

- API Key 或 Token 鉴权
- 高风险操作要求更高权限
- 配置与管理接口不得匿名开放

### 4.3 数据保护

- 传输使用 TLS
- 敏感数据避免明文落盘
- 密钥和令牌不得硬编码

### 4.4 审计与告警

- 关键操作必须留痕
- 异常行为应进入告警链路
- 审计信息应支持事后追踪

### 4.5 事件响应

- 按严重度分级处置
- 保留证据
- 提供隔离、限制和恢复策略

## 5. 开发要求

- 不在日志中打印敏感信息
- 不把调试后门提交到正式分支
- 新增接口必须说明鉴权和限流策略
- 变更安全行为必须同步更新安全文档

## 6. 合规要求

- 中国个人信息保护法
- GDPR
- 如涉及特定行业场景，再补充 HIPAA 等行业规范
