# 代码评审流程（Code Review Process）

版本：v2.0
最后更新：2026-03-16

## 1. 目标

- 保证代码质量与可维护性
- 降低缺陷与安全风险
- 保持架构一致性与风格统一

## 2. 流程概览

1. 提交 PR
- 描述背景、改动、影响范围
- 附带测试结果与截图（如有 UI）

2. 自动检查
- 单元测试
- Lint/格式化
- 安全扫描

3. 人工评审
- 功能正确性
- 架构边界
- 性能影响
- 安全风险

4. 合并与发布

## 3. 评审角色

- 主要评审人：模块负责人
- 次级评审人：跨模块代表
- 安全评审人：安全关键改动必须参与

## 4. 评审清单

- 是否符合层级依赖规范
- 是否有充足测试
- 是否正确处理异常
- 是否更新文档或说明
- 是否影响性能或安全

## 5. 严重度分类

- P0：生产事故风险
- P1：功能错误
- P2：性能/可维护性问题
- P3：风格或小问题
