# 编码规范（Coding Standards）

版本：v2.0
最后更新：2026-03-16

## 1. Python 基本规范

- 遵循 PEP8
- 使用类型注解
- 每个模块必须有 docstring

## 2. 命名规范

- 类名：PascalCase
- 函数/变量：snake_case
- 常量：UPPER_CASE

## 3. 错误处理

- 所有外部 IO 必须捕获异常
- 不吞掉异常，必须记录日志
- 业务异常使用自定义异常类型

## 4. 日志规范

- 使用 `logging`
- 生产环境禁止输出敏感信息
- 日志级别需区分 INFO/WARN/ERROR

## 5. 性能规范

- 避免阻塞操作在主线程
- 对高频接口使用缓存

## 6. 文档规范

- 公开接口必须有 docstring
- 新功能需更新 `docs/`
