# Mirexs 项目技术文档总览

## 0. 实现对齐摘要（2026-03-30）

本文档是 `docs/` 目录的总入口。当前状态仍为 `partial`，原因不是“没有文档”，而是仓库中仍同时存在三类内容：

1. 已经形成较稳定代码入口的文档
2. 已有设计和目录骨架、但实现仍在持续补齐的文档
3. 历史草案、内部研究和归档性材料

因此，阅读文档时必须先区分“开发契约文档”和“背景/研究/归档文档”。

## 1. 使用方式

如果你的目标是“根据文档开展后续开发”，建议按以下顺序阅读：

1. `README.md`：理解项目定位与功能全貌
2. `requirements.txt`：理解依赖边界与技术栈
3. `docs/architecture/overview.md`：理解系统分层
4. `docs/developer_guide/`：理解开发规则和工作方式
5. `docs/technical_specifications/`：理解接口、存储、性能和安全约束
6. 再进入具体子模块文档

## 2. 文档结构总览

`docs` 目录是 Mirexs 项目所有技术、业务、法律和用户相关文档的集中存储区域。其结构设计旨在清晰地划分不同领域的文档，便于检索和维护。主要子目录及其职责如下：

| 目录名称 | 主要内容 | 核心关注点 |
| :------- | :------- | :--------- |
| `api_reference/` | API、SDK、调用示例 | 对接和集成 |
| `architecture/` | 分层架构、模块设计、技术方案 | 开发设计依据 |
| `business/` | 商业计划、路线图、融资材料 | 商业规划 |
| `developer_guide/` | 环境、规范、评审、测试 | 日常开发依据 |
| `internal_docs/` | 研究、纪要、设计决策 | 内部沉淀与历史上下文 |
| `legal/` | 许可、服务条款、专利草案 | 法务合规 |
| `marketing/` | 产品叙事、定价、用户画像 | 市场定位 |
| `security/` | 审计、隐私、事件响应、合规 | 安全治理 |
| `technical_specifications/` | 技术规范、系统要求、性能、安全 | 实施与验收依据 |
| `user_manual/` | 安装、上手、功能、排障 | 用户侧说明 |

## 3. 如何判断一份文档能否直接指导开发

阅读任何一份文档前，建议先检查以下四点：

1. 是否声明 `status`
2. 是否给出真实代码路径
3. 是否区分“现状”和“目标态”
4. 是否写了输入、输出、依赖和验收标准

如果以上信息缺失，这份文档只能作为背景材料，不能直接作为开发契约。

## 6. 核心技术设计文档 (`architecture/`)

该目录包含了 Mirexs 项目最核心的 AI 架构和技术选型文档，是理解项目技术基石的关键。

*   **`ai_algorithms_handbook.md`**：详细介绍了 Mirexs 中使用的各类 AI 算法，包括 RAG 检索算法、强化学习奖励函数设计、多模型路由启发式算法等，是理解智能决策逻辑的参考手册。
*   **`big_data_models.md`**：阐述了 Mirexs 在大型语言模型（LLM）方面的选型策略（如 Llama 3.1, Qwen 3.5, DeepSeek V3）、量化技术（GGUF, EXL2）、微调方法（LoRA, QLoRA）以及模型并行推理机制，确保了模型的高效运行与定制化能力。
*   **`deep_thinking_engine.md`**：深入剖析了 Mirexs 的“深度思考”机制，包括思维链 (CoT) 实现、自我反思循环、多路径推理验证及其与知识图谱的协同，是 Mirexs 认知能力的核心体现。
*   **`emotion_nn.md`**：详细描述了情绪识别神经网络的具体架构（如 BiLSTM + Attention）、特征工程、训练策略及评估指标，是 Mirexs 情感感知能力的技术支撑。
*   **`fact_checker.md`**：深化了事实核查器的技术设计，涵盖信源评估、交叉验证与冲突解决机制，确保 Mirexs 输出信息的准确性和可靠性。
*   **`knowledge_graph.md`**：阐述了知识图谱的构建、存储（如 Neo4j/JanusGraph）、查询和更新机制，以及其在 Mirexs 语义记忆和推理中的作用。
*   **`multi_model_routing.md`**：详细设计了多模型路由策略，说明了如何根据任务类型、资源成本和性能要求智能调度不同的 AI 模型，以实现高效、精准的响应。
*   **`neural_networks_detail.md`**：提供了 Mirexs 中使用的各类神经网络（除情绪识别外）的详细设计，包括其结构、训练数据、优化方法等。
*   **`overview.md`**：Mirexs 整体架构的宏观视图，概述了感知层、认知核心层、行为生成层和交互呈现层的功能与相互关系。
*   **`proactive_behavior.md`**：设计了 Mirexs 主动行为引擎的机制，包括用户意图预测、情境感知和行为生成策略，使其能够提供更智能、更自然的交互体验。
*   **`reinforcement_learner.md`**：描述了强化学习在 Mirexs 中的应用，特别是如何通过奖励函数设计和策略学习来优化模型的决策和行为。
*   **`security_architecture.md`**：概述了 Mirexs 的安全架构，包括数据加密、访问控制、身份认证等安全机制，保障系统和数据的安全。

## 7. 技术规范文档 (`technical_specifications/`)

该目录包含了 Mirexs 项目各项技术实现的具体规范和标准。

*   **`api_specification.md`**：详细定义了 Mirexs 内部和外部 API 的接口规范，包括请求/响应格式、认证机制、错误码等。
*   **`compatibility_matrix.md`**：列出了 Mirexs 系统各组件、依赖库和硬件环境的兼容性要求。
*   **`data_pipeline_design.md`**：设计了从原始数据摄取、清洗、转换到特征工程、知识图谱构建和向量嵌入生成的完整大数据处理链路。
*   **`database_schema.md`**：详细定义了 Mirexs 项目所使用的各类数据库（关系型、NoSQL、向量数据库、图数据库）的表结构和数据模型。
*   **`distributed_inference_spec.md`**：制定了多卡/多节点环境下的分布式推理规范，包括负载均衡、模型分片、并行计算与容错机制。
*   **`performance_benchmarks.md`**：记录了 Mirexs 各核心模块的性能基准测试结果，包括延迟、吞吐量、资源占用等指标。
*   **`security_specification.md`**：详细定义了 Mirexs 的安全技术规范，包括数据加密标准、漏洞管理流程、安全审计要求等。
*   **`system_requirements.md`**：列出了 Mirexs 系统运行所需的软硬件环境要求。
