# Mirexs Issue 提交指南（ISSUES）

**最后更新**：2026-03-23

本文件用于补齐 README 中的入口链接，并定义最小的信息收集标准，便于快速定位问题。

## 1. Bug 报告（必填信息）

请在 Issue 中包含：

- 期望行为 vs 实际行为
- 复现步骤（尽量最小化）
- 环境信息：
  - OS 与版本
  - Python 版本（如相关）
  - GPU/显存（如相关）
  - 关键配置（请脱敏）：例如 `config/system/service_configs/api_config.yaml` 的相关段
- 日志与错误栈（脱敏后粘贴）
- 如涉及接口：请求/响应示例（注意脱敏）

## 2. 功能需求（建议结构）

- 需求背景：解决什么问题
- 使用场景：谁在什么情况下用
- 约束条件：隐私/性能/离线/成本
- 验收标准：可量化指标或可观察行为
- 影响范围：哪些模块/文档需要同步修改

## 3. 文档问题（强烈建议）

如果 Issue 指向文档错误，请包含：

- 错误文档路径与具体段落
- 与代码/配置冲突的证据（文件路径或配置字段）
- 建议修订口径（如 API base path、响应 Envelope 等）
