# Mirexs 隐私政策（Privacy Policy）

**版本：v2.0.0**  
**最后更新：2026-03-17**  
**作者：Zikang.Li**  
**适用范围**：Mirexs v2.0 本地优先数字生命体系统（包括核心引擎、SDK、REST API、插件系统）  
**生效日期**：2026年3月17日  
**位置相关**：本政策遵守香港《个人资料（私隐）条例》（Cap. 486）及相关国际隐私标准（GDPR 参考原则）

## 1. 承诺与原则

Mirexs 的核心设计理念是**“本地优先、用户主权、零云端默认”**。我们承诺：

- 您的所有对话、情绪数据、知识图谱、个性化模型微调**默认 100% 本地存储**，不上传至任何远程服务器。
- 除非您**明确授权**并启用可选云端增强功能（如联邦学习或实时知识接入），否则**无任何数据离开您的设备**。
- 我们不收集、不出售、不用于广告的任何个人数据。
- 您拥有**完全控制权**：随时导出、删除、重置所有个人数据。

## 2. 我们收集的数据类型（本地存储）

| 数据类型                  | 存储位置（本地）                  | 用途                                      | 是否可删除/导出 |
|---------------------------|-----------------------------------|-------------------------------------------|-----------------|
| 对话历史（文本/语音转录） | data/sessions/ 或 SQLite          | 上下文记忆、情绪分析、知识图谱构建        | 是（一键导出/删除） |
| 情绪记录（EmotionNN 输出）| data/emotion_history/ 或 KG       | 个性化情绪模型微调、强化学习奖励信号      | 是              |
| 知识图谱子图（您的实体/关系）| Neo4j 本地实例 (data/neo4j/)     | 长期记忆、主动行为触发                    | 是（子图导出/清空） |
| 强化学习 Q 值 / 过渡数据 | data/rl_qtable.db                 | 行为优化                                  | 是（可重置 RL） |
| 3D 头像行为日志           | data/avatar_logs/                 | 动画同步                                  | 是（临时日志）  |
| 审计日志（安全相关）      | data/audit.db（加密）             | 不可篡改记录所有决策/输入过滤             | 部分可导出      |
| 配置与偏好                | config/user_prefs.yaml            | 系统个性化                                | 是              |

**重要**：以上所有数据**仅存储在您的设备**（硬盘/SSD）。无云端备份、无同步服务。

## 3. 可选云端功能（需明确授权）

以下功能**默认关闭**，启用需在 config.yaml 中设置 `cloud_enhancements.enabled: true` 并提供授权 token：

- 实时知识接入（RSS/Web 爬取）→ 可能发送查询关键词（匿名化）
- 联邦学习微调（情绪/强化模型）→ 仅上传匿名梯度（差分隐私保护）
- 远程 API 模式（非本地）→ 对话内容可能传输至您指定的服务器

**授权流程**：
1. 首次启用时弹出确认对话（CLI/UI）
2. 记录在 audit.log 中（不可删除）
3. 可随时在 config 中关闭并删除相关缓存

## 4. 数据处理与保护措施

- **加密**：所有本地数据文件使用 AES-256-GCM 加密（密钥由用户设备生成或 config 指定）
- **匿名化**：用户 ID 默认使用本地 hash（SHA-256 + salt），不含真实身份信息
- **访问控制**：插件/模块仅能访问必要数据（最小权限原则，参考 security_architecture.md）
- **审计**：每条关键操作（输入、路由决策、情绪微调、KG 更新）记录不可篡改日志
- **数据删除**：支持“一键重置”：
  - 删除所有 sessions/、emotion_history/、rl_qtable.db
  - 清空 Neo4j 个人子图（MATCH (n) WHERE n.user_id = $hash DETACH DELETE n）
  - 重置模型 adapter（LoRA 文件）

## 5. 您的数据权利（香港私隐条例对应）

| 权利（英文）              | 对应权利（中文）          | 如何行使                                      |
|---------------------------|---------------------------|-----------------------------------------------|
| Right to Access           | 查阅个人资料              | 通过 SDK client.export_all_data() 或 CLI     |
| Right to Rectification    | 更正不准确资料            | 情绪/知识手动纠正（emotion/feedback, memory/add） |
| Right to Erasure          | 删除个人资料              | client.reset_user_data() 或 config 重置       |
| Right to Restrict Processing | 限制处理                | 关闭 proactive/rl/emotion 模块                |
| Right to Data Portability | 资料可携权               | 导出 JSONL/CSV 格式（对话、KG、情绪记录）     |
| Right to Object           | 反对处理                  | 禁用任何可选云端功能                          |

行使权利无费用，响应时间 ≤ 30 天（本地即时）。

## 6. 儿童隐私

Mirexs 不针对 13 岁以下儿童设计。如检测到用户年龄 < 13（通过自我声明或情绪上下文），系统将：
- 限制记忆功能
- 禁用强化学习微调
- 弹出监护人确认提示

## 7. 政策更新

- 本政策可能因功能更新而变更
- 重大变更（如引入云端默认）将通过 CLI/UI 弹窗通知
- 继续使用即视为接受最新版本
