## 0. 实现对齐摘要（2026-03-23）

本文为**隐私承诺与系统约束**的政策性文档；为避免“文档宣称”与“仓库现实”不一致，列出当前仓库可核验的对齐点与差异：

- **隐私偏好配置（可核验）**：`config/user/preferences/privacy.yaml`
- **隐私保护实现（可核验）**：`security/privacy_protection/`（同意管理、匿名化、数据加密、保留策略等）
- **用户数据管理入口（可核验）**：`data/user_data/` 与 `data/databases/`
- **差异说明**：联网抓取/联邦学习/远程服务等能力在仓库中存在配置与目录骨架，但端到端链路仍需补齐；如后续引入任何默认联网/云端行为，必须先更新本文档并通过用户显式授权。

## 1. 承诺与原则

Mirexs 的核心设计理念是**“本地优先、用户主权、零云端默认”**。我们承诺：

- 您的所有对话、情绪数据、知识图谱、个性化模型微调**默认 100% 本地存储**，不上传至任何远程服务器。
- 除非您**明确授权**并启用可选云端增强功能（如联邦学习或实时知识接入），否则**无任何数据离开您的设备**。
- 我们不出售、不用于广告的任何个人数据；若您在隐私偏好中启用“分析/共享”类选项（见第 3 节），其范围必须被限制在“最小必要、匿名化/聚合、可随时关闭”。
- 您拥有**完全控制权**：随时导出、删除、重置所有个人数据。

## 2. 我们收集的数据类型（本地存储）

| 数据类型                  | 存储位置（本地）                  | 用途                                      | 是否可删除/导出 |
|---------------------------|-----------------------------------|-------------------------------------------|-----------------|
| 对话历史（文本/语音转录） | `data/user_data/`（实现与配置决定具体落盘） | 上下文记忆、情绪分析、知识图谱构建        | 是（导出/删除能力需由上层产品入口提供） |
| 情绪记录（模型输出/反馈） | `data/user_data/learning_data.py`（及相关落盘） | 个性化适配、学习反馈信号                  | 是              |
| 知识图谱（实体/关系）     | `data/databases/graph_db/`（可选 Neo4j 适配：`data/databases/graph_db/neo4j_integration.py`） | 长期记忆、检索与推理                      | 是（按实现提供导出/清空） |
| 强化学习模型/训练历史     | `data/reinforcement_learning/`（见 `cognitive/learning/reinforcement_learner.py` 默认值） | 行为优化                                  | 是（可重置/清空） |
| 3D 头像行为/渲染运行数据  | 运行时日志/缓存（常见落盘在 `data/` 下，由配置与实现决定） | 动画同步、性能诊断                        | 是（临时数据可清理） |
| 审计日志（安全相关）      | `data/security/audit/`（见 `security/security_monitoring/audit_logger.py` 默认值） | 安全审计、追溯与取证                      | 可导出；可按保留策略归档/清理 |
| 配置与偏好                | `config/user/preferences/*.yaml`、`config/user/personalization/*.yaml`、`config/system/*.yaml` | 系统个性化与功能开关                      | 是              |

**重要**：以上数据**默认仅存储在您的设备**（硬盘/SSD）。如启用联网/云端能力（见第 3 节），将发生受控的网络传输与/或远端存储，并需保留可审计证据链。

## 3. 可选云端功能（需明确授权）

以下功能（或规划功能）**默认关闭**；启用需用户显式授权，并在隐私偏好与系统配置中打开相关开关（参考 `config/user/preferences/privacy.yaml` 与 `config/system/main_config.yaml` 中的功能开关口径），必要时还需配置凭据/token：

- 实时知识接入（RSS/Web 爬取）→ 可能发送查询关键词（匿名化）
- 联邦学习/远端训练（规划）→ 若启用，仅上传匿名化/聚合后的更新（需差分隐私等保护）
- 云端备份（规划/可选）→ 若启用，仅备份您授权的内容与范围（默认应关闭）
- 远程 API 模式（规划/可选）→ 若启用，对话内容可能传输至您指定的服务器/网关

**授权流程**：
1. 首次启用时弹出确认对话（UI/CLI 由产品形态决定）
2. 记录在审计日志中（如启用审计：见第 2 节“审计日志”）
3. 可随时关闭并删除相关缓存/索引（删除粒度与路径以实现为准）

## 4. 数据处理与保护措施

- **加密**：支持 AES‑256‑GCM 等算法对静态数据加密（实现参考 `security/privacy_protection/data_encryption.py`；密钥管理参考 `security/access_control/key_management.py`）
- **匿名化**：用户 ID 默认使用本地 hash（SHA-256 + salt），不含真实身份信息
- **访问控制**：插件/模块仅能访问必要数据（最小权限原则，参考 security_architecture.md）
- **审计**：每条关键操作（输入、路由决策、情绪微调、KG 更新）记录不可篡改日志
- **数据删除**：支持“一键重置”：
  - 清理用户数据与学习数据（典型落盘位于 `data/` 下，具体路径以实现与配置为准）
  - 若启用了 Neo4j 等外部数据库后端，支持按用户标识清空个人子图（示例：`MATCH (n) WHERE n.user_id = $hash DETACH DELETE n`）
  - 重置强化学习训练历史与模型（典型落盘：`data/reinforcement_learning/`）
  - 重置模型 adapter（如 LoRA 等，具体目录以模型管理实现为准）

## 5. 您的数据权利（香港私隐条例对应）

| 权利（英文）              | 对应权利（中文）          | 如何行使                                      |
|---------------------------|---------------------------|-----------------------------------------------|
| Right to Access           | 查阅个人资料              | 通过导出/备份能力（参考 `data/user_data/backup_manager.py`）或产品侧 UI/CLI（规划） |
| Right to Rectification    | 更正不准确资料            | 通过用户纠错入口（知识/情绪/偏好纠错；入口形态由产品决定） |
| Right to Erasure          | 删除个人资料              | 通过“一键重置/清理本地数据”能力（路径与粒度以实现为准） |
| Right to Restrict Processing | 限制处理                | 关闭 proactive/rl/emotion 模块                |
| Right to Data Portability | 资料可携权               | 导出 JSONL/CSV 格式（对话、KG、情绪记录）     |
| Right to Object           | 反对处理                  | 禁用任何可选云端功能                          |

行使权利无费用，响应时间 ≤ 30 天（本地即时）。

## 6. 儿童隐私

Mirexs 不针对 16 岁以下儿童设计。如检测到用户年龄 < 16（通过自我声明或情绪上下文），系统将：
- 限制记忆功能
- 禁用强化学习微调
- 弹出监护人确认提示

## 7. 政策更新

- 本政策可能因功能更新而变更
- 重大变更（如引入云端默认）将通过 CLI/UI 弹窗通知
- 继续使用即视为接受最新版本
