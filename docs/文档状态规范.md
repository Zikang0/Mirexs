# Mirexs 文档状态规范（2026.03）

每份 .md 文件头部必须添加以下 YAML front-matter：

```yaml
---
status: planned | partial | implemented | deprecated | archived
last_reviewed: 2026-03-26
corresponds_to_code: "真实相对路径或 '暂无'"
related_issues: ""
---
```

状态定义：
- planned：仅规划，代码未开始
- partial：部分实现，与代码存在偏差
- implemented：文档与代码完全一致
- deprecated/archived：已废弃或归档
