# 知识图谱（Knowledge Graph）架构规范

版本：v2.0
最后更新：2026-03-16

## 1. 目标与定位

知识图谱用于让 Mirexs 具备“关系理解”能力，不只是记住事实，还能理解事实之间的关系、时间与强度，从而驱动推荐、推理与长期记忆。

核心目标：
- 结构化存储“人‑事‑物‑地‑概念”的关系。
- 支持多跳推理与可解释结果。
- 与向量记忆协同，提升检索与关联质量。

## 2. 数据模型

### 2.1 实体（Entity）

- `id`：全局唯一标识。
- `name`：名称/别名。
- `type`：实体类型（人/物/地点/概念/事件）。
- `properties`：属性键值对（如标签、描述、时间）。
- `confidence`：置信度。
- `created_time / updated_time`：时间戳。

### 2.2 关系（Relation）

- `id`：全局唯一标识。
- `source_entity / target_entity`：起止实体。
- `relation_type`：关系类型（喜欢/讨厌/属于/发生在/拥有）。
- `properties`：关系属性（强度、频率、时间范围）。
- `confidence`：置信度。

### 2.3 证据（Evidence，可选）

- `source`：证据来源（对话/文件/网页）。
- `snippet`：证据片段。
- `weight`：证据权重。
- `timestamp`：证据时间。

## 3. 存储策略

- 持久层：Neo4j（生产环境）
- 内存层：NetworkX（快速遍历）
- 向量层：Chroma/FAISS（语义检索）

## 4. 建图流程

1. 实体抽取
- 从对话、文档、网页中识别实体。

2. 关系抽取
- 基于规则与分类器识别关系类型。

3. 实体消歧
- 解决别名、重复实体。

4. 置信度评估
- 根据来源可信度与出现频次计算。

5. 入库与索引
- 写入图数据库与向量库。

## 5. 查询与推理

### 5.1 查询能力

- 实体检索：按名称/类型
- 关系检索：按类型/方向
- 子图提取：中心实体 + 半径

### 5.2 推理能力

- 多跳推理：支持 2‑3 跳关系路径
- 关系权重评分：综合强度与置信度

## 6. 一致性与冲突处理

- 多来源冲突：以“置信度 + 新鲜度”排序
- 用户纠错：用户声明优先级最高
- 历史保留：旧事实不删除，仅降权

## 7. 与记忆系统的协同

- 情景记忆：记录事件
- 语义记忆：抽象为图谱
- 程序记忆：流程关联到任务节点

## 8. API 契约

- `add_entity(name, type, properties)`
- `add_relation(source_id, target_id, relation_type)`
- `search_entities(query)`
- `infer_relations(entity_id, max_depth)`

## 9. 性能指标

- 100 万节点查询 < 100ms
- 2 跳推理 < 300ms
