# AI 算法手册技术设计

## 1. 引言

本文档旨在汇总 Mirexs 项目中采用的关键 AI 算法及其技术设计，涵盖 RAG（检索增强生成）的检索算法、强化学习（RL）的奖励函数设计、多模型路由的启发式算法、以及其他核心决策算法。这些算法共同构成了 Mirexs 智能决策和行为的基础，确保其能够高效、准确地处理复杂任务并与用户进行情感化交互。

## 2. RAG（检索增强生成）的检索算法

Mirexs 采用 RAG 架构来实现实时知识更新和减少大型语言模型（LLM）的“幻觉”问题。其核心在于高效、准确地从海量知识库中检索相关信息。

### 2.1 检索流程

1.  **用户查询/LLM 生成中间查询**：当用户提出问题或 LLM 在生成回复过程中需要外部知识时，会生成一个查询。
2.  **查询向量化**：使用预训练的嵌入模型（如 `bge-small-zh-v1.5` 或 `text-embedding-ada-002`）将查询转换为高维向量。
3.  **向量相似度搜索**：在向量数据库（如 Chroma 或 FAISS）中，通过计算查询向量与知识库中所有文档块（chunk）向量的相似度（如余弦相似度），找出最相关的 K 个文档块。
4.  **重排序 (Re-ranking)**：对初步检索到的文档块进行二次排序，通常使用交叉编码器（Cross-Encoder）模型，它能更精细地评估查询与文档块之间的语义相关性，从而提升检索质量。
5.  **上下文构建**：将重排序后的顶部文档块作为上下文，与原始查询一起输入给 LLM 进行生成。

### 2.2 关键算法与技术

*   **嵌入模型 (Embedding Models)**：
    *   **选型**：优先选择在特定领域（如中文、通用知识）表现优异且推理效率高的模型。例如，`bge-small-zh-v1.5` 在中文语义匹配任务上表现出色，且模型体积小。
    *   **更新机制**：定期更新嵌入模型，以适应新的语言模式和知识。
*   **向量数据库 (Vector Databases)**：
    *   **Chroma**：轻量级、易于部署，适合中小型知识库和本地部署场景。
    *   **FAISS (Facebook AI Similarity Search)**：高性能的相似度搜索库，适合大规模向量索引和高并发查询。
    *   **索引策略**：采用 HNSW (Hierarchical Navigable Small World) 等高效索引算法，加速相似度搜索。
*   **重排序算法**：
    *   **交叉编码器 (Cross-Encoder)**：将查询和文档块拼接后输入一个小型 Transformer 模型，直接预测相关性得分。相比双编码器（Dual-Encoder），其计算成本更高但准确率更高。
    *   **多样性重排序**：在保证相关性的前提下，引入多样性度量，避免检索结果过于集中于相似内容。

## 3. 强化学习（RL）的奖励函数设计

Mirexs 的强化学习模块（`reinforcement_learner.py`）通过与用户的交互和环境反馈来优化其行为策略。奖励函数的设计是 RL 成功的核心。

### 3.1 奖励函数目标

*   **促进用户满意度**：鼓励生成用户喜欢、有帮助的回复和行为。
*   **维持情感一致性**：奖励与用户情绪状态相匹配的交互。
*   **鼓励探索与学习**：在一定程度上奖励探索新行为和知识。
*   **惩罚不安全/不当行为**：对可能引起用户不适或违反安全原则的行为给予负面奖励。

### 3.2 奖励函数组成

Mirexs 采用复合奖励函数，结合了多种信号：

*   **显式奖励 (Explicit Rewards)**：
    *   **用户反馈**：直接来自用户的点赞、评分、纠正等（例如：用户点赞 +1.0，用户纠正 -0.5）。
    *   **任务完成度**：成功完成用户请求的任务（例如：成功预订机票 +2.0）。
*   **隐式奖励 (Implicit Rewards)**：
    *   **对话轮次**：对话轮次越少完成任务，奖励越高（例如：`1 / log(turns)`）。
    *   **情感匹配度**：Mirexs 的回复情感与用户当前情感的匹配程度（例如：积极情感匹配 +0.2）。
    *   **知识图谱更新**：成功提取并更新知识图谱中的新实体或关系（例如：每次成功更新 +0.1）。
    *   **工具调用成功率**：成功调用外部工具并获得预期结果（例如：成功调用 API +0.3）。
*   **惩罚项 (Penalties)**：
    *   **安全违规**：触发敏感词、生成不当内容（例如：-5.0）。
    *   **任务失败**：未能完成用户请求（例如：-1.0）。
    *   **重复/无效行为**：生成重复信息或无意义的回复（例如：-0.1）。

### 3.3 奖励整形 (Reward Shaping)

为了加速学习过程，Mirexs 采用奖励整形技术，提供额外的、与任务目标相关的中间奖励信号。例如：

*   **好奇心奖励**：鼓励模型探索未曾访问过的状态-动作对，或提及新颖的话题。
*   **关系强度奖励**：如果行为有助于提升与用户的关系强度（通过知识图谱中的关系属性衡量），则给予额外奖励。

## 4. 多模型路由的启发式算法

Mirexs 的 `multi_model_router.py` 模块负责根据任务类型、复杂度和可用硬件资源动态选择最合适的 LLM。这主要依赖一套启发式算法。

### 4.1 路由决策因子

1.  **任务类型**：
    *   **日常聊天/情感陪伴**：优先选择通用性强、响应速度快、情感理解能力好的模型（如 Llama 3.1 70B, Qwen 3.5 32B）。
    *   **编程/复杂推理**：优先选择在逻辑推理和代码生成方面表现优异的模型（如 DeepSeek V3, Qwen3-Coder-32B）。
    *   **长上下文处理**：优先选择具有超长上下文窗口的模型（如 Llama 4 Maverick, Qwen3-Next）。
    *   **多模态任务**：优先选择多模态融合能力强的模型（如 Llama 4 Scout, Qwen3-Omni）。
2.  **任务复杂度**：
    *   通过 `task_decomposer.py` 和 `problem_analyzer.py` 评估任务的复杂性（例如：涉及的实体数量、推理步骤、所需外部工具调用次数）。
    *   简单任务：可使用轻量级模型。
    *   复杂任务：倾向于使用能力更强、参数量更大的模型。
3.  **硬件资源**：
    *   `hardware_detector.py` 提供实时的 GPU 显存、CPU 负载等信息。
    *   根据模型所需的 VRAM 和当前可用资源进行匹配。如果资源不足，则降级到量化模型或更小的模型。
4.  **用户偏好**：
    *   结合用户在 `preferences.py` 中设置的模型偏好。
    *   通过强化学习学习到的用户对不同模型输出的偏好。

### 4.2 路由算法流程

1.  **预处理**：对用户输入进行初步分析，识别任务类型和初步复杂度。
2.  **候选模型筛选**：根据任务类型和硬件资源，从 `model_configs.yaml` 中筛选出符合条件的候选模型列表。
3.  **优先级排序**：
    *   **硬性约束**：首先排除不满足 VRAM 或上下文窗口要求的模型。
    *   **软性偏好**：根据任务复杂度、用户偏好、模型历史表现（如成功率、延迟）等因素，对候选模型进行加权评分。
    *   **动态调整**：结合实时负载情况，对评分进行微调，避免单一模型过载。
4.  **模型选择**：选择评分最高的模型进行推理。如果最高评分模型不可用（例如，正在加载中），则选择次优模型。
5.  **Fallback 机制**：如果所有模型都无法满足要求，则降级到默认的轻量级模型或向用户请求澄清。

## 5. 其他核心决策算法

### 5.1 知识图谱推理算法

*   **路径查找算法**：使用 Cypher 查询语言结合图算法（如 BFS, DFS, Dijkstra）在 Neo4j 中查找实体间的关系路径。
*   **嵌入相似度推理**：通过向量相似度搜索，发现语义上相关的实体或概念，即使它们之间没有直接的图谱连接。
*   **时间衰减与重要性计算**：对知识图谱中的节点和关系引入时间衰减因子和重要性评分，确保知识的实时性和相关性。

### 5.2 行为规划算法

*   **分层任务规划**：`hierarchical_planner.py` 将高层目标分解为一系列可执行的原子操作，形成一个行为树或规划图。
*   **状态空间搜索**：在复杂任务中，可能需要使用 A* 或其他启发式搜索算法来寻找最优的行为序列。
*   **预警与主动行为**：`proactive_behavior.md` 中描述的引擎会基于用户情绪、知识图谱中的事件和预设规则，触发主动行为，例如提醒、建议或提供帮助。
