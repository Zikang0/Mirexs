## 1. 引言

Mirexs 的核心目标之一是超越传统 AI 的被动响应，实现主动的、有深度的“思考”。本文档详细阐述 Mirexs 深度思考引擎的技术设计，包括思维链（Chain-of-Thought, CoT）、自我反思（Self-Reflection）、多路径推理与验证，以及其与知识图谱和记忆系统的协同工作机制。深度思考引擎是 Mirexs 认知核心层（Cognitive Layer）的关键组成部分，旨在提升其处理复杂问题、进行逻辑推理和生成高质量决策的能力。

## 2. 核心设计理念

Mirexs 的深度思考引擎借鉴人类认知过程，将复杂问题分解为一系列可管理的中间步骤，并通过迭代和反馈机制不断优化思考过程和结果。其核心理念包括：

*   **透明性**：思考过程可追溯、可解释，而非黑箱操作。
*   **可控性**：能够根据任务需求和用户偏好调整思考的深度和广度。
*   **鲁棒性**：在面对不确定性或信息不足时，能够进行有效的推理和决策。
*   **持续优化**：通过自我反思和强化学习机制，不断提升思考能力。

## 3. 思维链（Chain-of-Thought, CoT）实现

思维链（CoT）是一种提示工程技术，通过引导大型语言模型（LLM）逐步推理，从而提高其在复杂任务上的表现。Mirexs 将 CoT 集成到其任务分解与规划模块中。

### 3.1 CoT 工作流程

1.  **任务接收**：用户输入一个复杂任务（例如：“帮我策划一个包含预算和行程的周末旅行计划”）。
2.  **初始分解**：`cognitive/reasoning/task_decomposer.py` 模块首先尝试将任务分解为高层次的子任务（例如：确定目的地、估算预算、规划行程、预订）。
3.  **CoT 提示生成**：对于每个子任务，深度思考引擎会生成一个包含 CoT 提示的指令，发送给选定的 LLM。例如，对于“估算预算”子任务，提示可能包含：“请逐步思考，列出估算周末旅行预算所需考虑的所有因素，然后根据这些因素给出详细的预算估算。”
4.  **LLM 逐步推理**：LLM 接收到 CoT 提示后，会输出详细的思考过程和最终答案。例如，它可能会列出交通、住宿、餐饮、景点门票等因素，并逐步计算。
5.  **结果解析与验证**：`cognitive/reasoning/problem_analyzer.py` 模块解析 LLM 的输出，提取关键信息，并进行初步的逻辑和事实验证。
6.  **迭代与细化**：如果初始结果不满意或存在歧义，引擎会生成新的 CoT 提示，要求 LLM 进一步细化或修正。

### 3.2 CoT 提示模板示例

```text
你是一个专业的规划助手。请逐步思考以下问题，并给出详细的解决方案。

**问题**：{用户问题}

**思考步骤**：
1. 首先，我会分析问题的核心需求和约束条件。
2. 接着，我会拆解问题为若干个子任务。
3. 对于每个子任务，我会考虑所有相关的因素和潜在的解决方案。
4. 我会评估这些方案的优缺点，并选择最优路径。
5. 最后，我会整合所有子任务的结果，形成最终的解决方案。

**解决方案**：
```

## 4. 自我反思（Self-Reflection）机制

自我反思机制允许 Mirexs 评估其自身的思考过程和输出结果，识别潜在错误或改进空间，并进行修正。这是一种迭代优化循环。

### 4.1 反思循环

1.  **初步输出**：CoT 过程生成一个初步的解决方案。
2.  **反思提示生成**：`cognitive/reasoning/self_reflection.py` 生成一个反思提示，要求 LLM 评估其自身的输出。例如：“请仔细检查上述旅行计划，是否存在逻辑漏洞、遗漏项或不合理之处？请提出改进建议。”
3.  **LLM 自我评估**：LLM 接收反思提示后，会对其先前的输出进行批判性分析，并提出改进意见。
4.  **修正与优化**：深度思考引擎根据 LLM 的反思结果，生成新的指令，引导 LLM 修正或优化解决方案。
5.  **迭代**：反思循环可以进行多次，直到达到预设的质量标准或迭代次数上限。

### 4.2 反思提示示例

```text
你刚刚生成了一个关于 {任务类型} 的计划。请你扮演一个严谨的审查者，从以下几个方面对该计划进行评估：

1.  **完整性**：是否遗漏了任何关键信息或步骤？
2.  **逻辑性**：计划的各个部分之间是否存在逻辑冲突或不连贯之处？
3.  **可行性**：在实际操作中，该计划是否容易实施？是否存在难以克服的障碍？
4.  **准确性**：计划中引用的事实或数据是否准确无误？
5.  **用户满意度**：如果我是用户，我对这个计划的哪些部分会感到满意，哪些部分可能不满意？

请列出所有发现的问题，并为每个问题提供具体的改进建议。如果计划完美，请说明。

**原计划**：
{LLM_Output_from_CoT}

**我的反思与改进建议**：
```

## 5. 多路径推理与验证

为了提高复杂推理的鲁棒性和准确性，Mirexs 采用多路径推理，即同时探索多个可能的推理路径，并通过交叉验证来选择最优解。

### 5.1 工作机制

1.  **并行 CoT**：对于关键或高风险的推理任务，深度思考引擎会并行启动多个 CoT 过程，每个过程可能使用略微不同的提示、不同的 LLM 模型（通过路由层，参见 `docs/architecture/multi_model_routing.md`；代码占位：`cognitive/reasoning/smart_router_integration.py`）或不同的初始假设。
2.  **结果聚合与比较**：`cognitive/reasoning/problem_analyzer.py` 模块聚合来自不同推理路径的结果。这可能包括：
    *   **一致性检查**：比较不同路径得出的结论是否一致。
    *   **证据链比对**：分析不同路径所依赖的证据和推理步骤。
    *   **置信度评估**：结合每个路径的 LLM 输出置信度、以及其所依赖的事实来源的可靠性，评估最终结论的置信度。
3.  **冲突解决**：如果不同路径得出相互矛盾的结论，引擎会触发一个专门的冲突解决机制，可能包括：
    *   **回溯与重新推理**：识别冲突点，并针对性地重新进行 CoT 和反思。
    *   **外部信息查询（可选）**：通过 `capabilities/knowledge/real_time_knowledge.py` 进行联网检索/摄取以获取更多证据（需遵循“本地优先、用户授权”的约束）。
    *   **用户介入**：在无法自动解决时，向用户寻求澄清或决策。

## 6. 与知识图谱和记忆系统的协同

深度思考引擎与 Mirexs 的知识图谱（Knowledge Graph）和混合记忆系统（Hybrid Memory）紧密集成，以提供更丰富、更准确的推理能力。

*   **知识图谱增强**：
    *   **事实检索**：在 CoT 过程中，深度思考引擎会频繁查询知识图谱，获取实体关系、属性和背景知识，确保推理基于准确的事实。
    *   **逻辑约束**：知识图谱中定义的本体和规则可以作为推理的逻辑约束，帮助 LLM 避免生成不符合常识或逻辑的答案。
    *   **推理验证**：LLM 生成的中间推理步骤和最终结论，可以通过与知识图谱中的已知事实进行比对来验证其正确性。
*   **记忆系统集成**：
    *   **情景记忆**：CoT 和自我反思过程中的关键中间状态、决策点和遇到的问题，会被存储到情景记忆中，以便后续回顾和学习。
    *   **语义记忆**：通过深度思考获得的新的概念、原理或推理模式，会被抽象并固化到语义记忆中，丰富 Mirexs 的长期知识。
    *   **工作记忆**：当前正在进行的思考任务的临时信息，存储在工作记忆中，供 CoT 和反思循环快速访问。

## 7. 性能指标与优化

*   **推理准确率**：通过多路径验证和自我反思，目标是复杂推理任务的准确率提升 15% 以上。
*   **思考深度**：衡量 CoT 步骤的平均数量和反思循环的次数。
*   **决策质量**：通过用户反馈和离线评估，持续优化决策的合理性和有效性。
*   **延迟优化**：通过并行推理、缓存机制和模型路由，控制深度思考过程的整体延迟。
